## Custom Project

### Dataset

- Dataset file: sales.csv
- Records: sales transactions
- Fields include: order_id, datetime, region_id, currency_code, product_id,
unit_price, quantity, discount_code, and customer/context fields
- Used the original sales dataset unchanged
- Reference datasets used: regions.csv for tax rates, discount_codes.csv for
discount percentages

### Data Contract

- Required fields: order_id, datetime, region_id, currency_code, product_id,
unit_price, quantity
- Optional field: discount_code
- Values must match reference data for IDs: valid region IDs and valid discount
codes
A message is valid if all required fields are present and lookups exist;
- invalid messages are rejected and skipped

### Kafka Messages

- Producer sends raw sales records from sales.csv
-Uses the project Kafka topic defined in settings
- Messages are sent with the original record fields, without altering the source
fields

### Consumer Validation

- Consumer checks required fields and looks up reference values
- Accepted messages are enriched and written to the output CSV
- Rejected or invalid messages are skipped and logged
- Validation protects analytics results by keeping only complete, referenced
records

### Data Engineering and Enrichment

- Added derived fields:
- subtotal
- discount_amount
- price_after_discount
- tax_amount
- total
- Uses reference data from regions.csv and discount_codes.csv
- Raw fields are transformed into financial amounts and discount-aware totals
- Changed calculation so tax is computed after discount is applied

### Streaming Analytics

- Tracks running summaries as messages arrive
- Summarized values:
- total sales
- average order value
- minimum order value
- maximum order value
- Running statistics update with each consumed message

### Experiments

- Phase 4 change: increased producer message count
- Phase 5 application: updated consumer pipeline to pass discount lookup through
validation and enrichment, then extended output schema

### Results

- Producer successfully sent messages
- Consumer successfully processed messages
- All source messages were accepted, with no discount applied because sample
records had empty discount_code
- Output CSV now contains the new derived fields
- Logs show validation, enrichment, and running total updates

### Interpretation

- The workflow now validates and enriches streaming sales data more thoroughly
- Learned that message validation is essential to keep bad data out of analytics
- Learned that enrichment can add business value by computing discount-aware
totals
- Running summaries show how sales performance evolves as orders are consumed
- This gives business insight into order values, discount impact, and validated
revenue totals
